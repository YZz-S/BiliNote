# BiliNote GUI启动器

## 概述

BiliNote GUI启动器是一个为BiliNote AI视频笔记项目设计的图形化启动和管理工具。它提供了直观的界面来管理前后端服务、配置项目参数、监控运行状态和查看系统信息。

## 功能特性

### 🚀 项目启动控制
- **独立启动**：可单独启动前端或后端服务
- **一键启动**：同时启动前后端服务
- **状态监控**：实时显示服务运行状态、PID、端口占用情况
- **进程管理**：支持启动、停止、重启操作
- **Conda支持**：自动识别并激活conda虚拟环境

### ⚙️ 配置管理
- **端口配置**：可视化编辑前后端端口设置
- **端口检查**：检测端口占用情况和冲突
- **.env文件编辑**：内置文本编辑器，支持语法高亮
- **配置验证**：自动验证配置项的有效性
- **配置备份**：自动创建配置文件备份

### 📊 日志监控
- **实时日志**：分别显示前端、后端和系统日志
- **日志过滤**：按级别过滤日志（ERROR、WARNING、INFO、DEBUG）
- **颜色标识**：不同级别的日志使用不同颜色显示
- **日志导出**：支持将日志导出为文本文件
- **错误诊断**：自动识别常见错误并提供解决建议

### 🔍 系统信息
- **依赖检查**：检查Python、Node.js、pnpm、FFmpeg等依赖
- **系统资源**：监控CPU、内存、磁盘使用情况
- **项目结构**：验证项目文件和目录的完整性
- **系统诊断**：自动诊断系统问题并提供建议

## 安装要求

### 必需依赖
- **Python 3.6+**：运行GUI启动器
- **tkinter**：GUI界面库（通常随Python安装）
- **psutil**：系统监控库

### 项目依赖
- **Node.js 14.0+**：前端开发环境
- **pnpm 6.0+**：包管理器
- **FFmpeg 4.0+**：音视频处理（可选）

### 安装psutil
```bash
pip install psutil
```

## 使用方法

### 快速开始

1. **确保在项目根目录**
   ```bash
   cd /path/to/BiliNote
   ```

2. **启动演示版GUI**（推荐首次使用）
   ```bash
   python start_gui.py
   ```

3. **启动完整版GUI**
   ```bash
   python gui_launcher.py
   ```

### 详细使用步骤

#### 1. 系统检查
- 启动GUI后，首先进入"系统信息"页面
- 点击"检查依赖"确保所需软件已安装
- 查看"项目结构"确认文件完整性

#### 2. 配置管理
- 进入"配置管理"页面
- 在"端口配置"标签页中设置前后端端口
- 使用".env文件编辑"标签页修改环境变量
- 点击"验证配置"检查设置是否正确

#### 3. 启动服务
- 返回"主控制台"页面
- 点击"启动后端"启动后端服务
- 等待后端启动完成后，点击"启动前端"
- 或直接点击"一键启动全部"同时启动两个服务

#### 4. 监控运行
- 在"主控制台"查看服务状态和资源使用情况
- 进入"日志监控"页面查看实时日志
- 使用日志过滤功能查看特定级别的日志

## 界面说明

### 主控制台
- **启动控制区域**：前后端服务的启动/停止按钮
- **状态监控表格**：显示服务状态、PID、端口、资源使用情况
- **快速访问按钮**：打开前端页面、后端API文档、项目目录

### 配置管理
- **端口配置**：设置BACKEND_PORT、FRONTEND_PORT、VITE_FRONTEND_PORT
- **.env文件编辑**：直接编辑配置文件内容
- **配置验证**：检查配置项的有效性和兼容性

### 日志监控
- **前端日志**：Vite开发服务器的输出
- **后端日志**：FastAPI服务器的输出
- **系统日志**：GUI启动器自身的日志

### 系统信息
- **依赖检查**：显示各个依赖的安装状态和版本
- **系统资源**：实时系统资源使用情况
- **项目结构**：项目文件和目录的检查结果

## 配置文件

### .env文件结构
```ini
# 端口配置
BACKEND_PORT=8483
FRONTEND_PORT=3015
VITE_FRONTEND_PORT=3015

# API配置
VITE_API_BASE_URL=http://127.0.0.1:8483
VITE_SCREENSHOT_BASE_URL=http://127.0.0.1:8483/static/screenshots

# 转写器配置
TRANSCRIBER_TYPE=fast-whisper
WHISPER_MODEL_SIZE=base

# Conda环境
CONDA_ENV_NAME=bilinote
```

### 重要配置项说明
- **BACKEND_PORT**：后端服务端口（默认8483）
- **VITE_FRONTEND_PORT**：前端开发服务器端口（默认5173）
- **CONDA_ENV_NAME**：conda虚拟环境名称（可选）
- **TRANSCRIBER_TYPE**：音频转写器类型

## 故障排除

### 常见问题

#### 1. 端口被占用
**现象**：启动服务时提示端口已被占用
**解决方案**：
- 在配置管理中检查端口占用情况
- 修改为其他可用端口
- 或停止占用端口的进程

#### 2. 依赖缺失
**现象**：系统信息显示某些依赖未安装
**解决方案**：
- 安装缺失的依赖软件
- 确保软件已添加到PATH环境变量

#### 3. 启动失败
**现象**：点击启动按钮后服务无法启动
**解决方案**：
- 检查日志监控页面的错误信息
- 确认项目结构完整
- 验证配置文件正确性

#### 4. GUI无法启动
**现象**：运行启动脚本时出现错误
**解决方案**：
- 确保Python版本3.6+
- 安装psutil：`pip install psutil`
- 检查是否在项目根目录运行

### 日志文件位置
- **配置备份**：`config_backups/`目录
- **导出日志**：用户指定位置

## 开发说明

### 项目结构
```
BiliNote/
├── gui_launcher.py          # 主GUI程序
├── start_gui.py             # 演示版启动器
├── test_modules.py          # 模块测试脚本
└── modules/                 # 功能模块
    ├── __init__.py
    ├── process_manager.py   # 进程管理
    ├── config_manager.py    # 配置管理
    ├── log_monitor.py       # 日志监控
    └── system_detector.py   # 系统检测
```

### 模块说明
- **ProcessManager**：负责前后端进程的启动、停止和监控
- **ConfigManager**：处理.env文件的读写和配置验证
- **LogMonitor**：实时监控和管理应用程序日志
- **SystemDetector**：检查系统依赖和资源状态
